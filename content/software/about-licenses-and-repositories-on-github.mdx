title: 'Sobre licenças e repositórios no GitHub'
publishedAt: '2023-04-12'
---

Neste artigo, vamos falar sobre licenças e repositórios no GitHub, uma plataforma online que permite
hospedar, gerenciar e colaborar em projetos de software.

O código do officia está
[disponível publicamente no GitHub](https://github.com/theduardomaciel/officia). Você pode acessar o
repositório e ver como o aplicativo foi desenvolvido, quais as tecnologias utilizadas, quais as
funcionalidades implementadas e quais os desafios enfrentados.

Você também pode baixar o código para o seu computador e executá-lo localmente, ou fazer alterações
e testes para aprender mais sobre programação móvel. No entanto, nenhuma licença foi definida para o
código do officia até o momento, o que significa que você não pode fazer uso comercial do código,
nem distribuir cópias do código sem consentimento, por enquanto.

O projeto todo foi desenvolvido por um único programador, que busca compartilhar seu pouco
conhecimento com outras pessoas que se interessem pelo tema. O objetivo é criar uma comunidade de
aprendizado e troca de experiências em torno do officia, incentivando a participação de novos
programadores que queiram contribuir com o desenvolvimento do aplicativo ou criar seus próprios
projetos baseados nele.

Entretanto, é importante respeitar a licença que acompanha o código do officia. A licença é um
documento legal que define os direitos e deveres dos usuários do software. No caso do officia, a
licença escolhida de maneira preliminar foi a **GNU General Public License v3.0 (GPL-3.0)**, que é
uma licença de código aberto que garante as seguintes liberdades:

-   A liberdade de executar o programa como você desejar, para qualquer propósito.
-   A liberdade de estudar como o programa funciona e adaptá-lo às suas necessidades.
-   A liberdade de redistribuir cópias do programa para ajudar outras pessoas.
-   A liberdade de melhorar o programa e liberar suas modificações para o público, de modo que toda
    a comunidade se beneficie.

No entanto, essas liberdades vêm acompanhadas de algumas obrigações:

-   Você deve manter a licença original do programa e dar crédito ao autor original.
-   Você deve disponibilizar o código-fonte de qualquer versão modificada do programa que você
    distribuir.
-   Você deve usar a mesma licença ou uma licença compatível para qualquer versão modificada do
    programa que você distribuir.
-   Você não pode usar o programa para propósitos comerciais, visto que o propósito da
    disponibilidade é puramente educacional.

Essas obrigações visam garantir que o software permaneça livre e aberto para todos, e que ninguém se
aproveite indevidamente do trabalho alheio. Se você quiser saber mais sobre a licença GPL-3.0, você
pode consultar o texto completo no seguinte endereço:
[https://www.gnu.org/licenses/gpl-3.0.html](https://www.gnu.org/licenses/gpl-3.0.html).

Esperamos que este artigo tenha sido útil para você entender melhor como funciona o GitHub e como
você pode usar o código do officia para aprender mais sobre programação móvel.
